# GenFeed.ai Documentation Guidelines

## Documentation Structure

The GenFeed.ai documentation is organized into the following sections:

### 1. Introduction (`intro.md`)
- Welcome message and platform overview
- Key features and quick navigation
- Getting started steps

### 2. Getting Started
- `getting-started.md` - Comprehensive onboarding guide
- `pricing.md` - Subscription plans and credit system

### 3. AI Models & Features
- `available-models.md` - Complete model catalog with specifications
- `models.md` - Model selection guide and comparisons

### 4. Content Creation
- `prompting-guide.md` - General prompting best practices
- `asset-prompting-guide.md` - Asset-specific prompting (images, videos, audio)
- `voice-recognition.md` - Voice input features
- `keyboard-shortcuts.md` - Productivity shortcuts

### 5. Publishing & Growth
- `social-media-setup.md` - Platform integration and publishing

### 6. Advanced
- `advanced-features.md` - API, automation, enterprise features
- `admin-guide.md` - Team and workspace management

### 7. Help & Support
- `faq.md` - Frequently asked questions

## Theme Configuration

The documentation uses a custom dark theme matching the GenFeed.ai app:
- Primary color: Indigo/Purple (#6366f1)
- Dark backgrounds: Black (#0a0a0a) to dark gray (#262626)
- High contrast for readability
- Smooth transitions between themes

## Content Guidelines

### Writing Style
- Clear and concise explanations
- Use examples and code snippets
- Include visual indicators (✅ ❌) for do's and don'ts
- Organize with clear headings and sections

### Technical Accuracy
- Keep model specifications up to date
- Verify API endpoints and parameters
- Test code examples before documenting
- Update credit costs as they change

### User Focus
- Address common use cases first
- Provide step-by-step instructions
- Include troubleshooting sections
- Link to related documentation

## Maintenance

### Regular Updates
- Review model availability monthly
- Update pricing information
- Add new features as released
- Remove deprecated content

### Quality Checks
- Test all links and references
- Verify code examples work
- Check for outdated screenshots
- Ensure consistent formatting

# Publisher App - Complete Guide

Publisher is your content scheduling and publishing hub. Schedule content across multiple platforms, use the calendar view, compose threads, and get AI-powered best time to post recommendations.

## Overview

Publisher helps you schedule, manage, and publish content across all your social media platforms. Plan your content calendar, compose multi-part threads, and optimize posting times for maximum engagement.

## Getting Started

### Accessing Publisher

1. Navigate to Publisher from main navigation
2. Or use command palette (`⌘K`) and type "publisher"
3. Or use keyboard shortcut `⌘3` (Mac) or `Ctrl+3` (Windows/Linux)

### Publisher Interface

- **Calendar View**: Visual schedule of all content
- **Content Library**: Browse ingredients to publish
- **Composer**: Create new content for publishing
- **Scheduled Posts**: View and manage scheduled content
- **Platform Selector**: Choose target platforms

## Calendar View

### Viewing Your Schedule

**Monthly View:**

- See entire month at a glance
- Color-coded by platform
- Click date to see scheduled posts
- Drag and drop to reschedule

**Weekly View:**

- Detailed weekly schedule
- See exact posting times
- Platform-specific columns
- Time zone indicators

**Daily View:**

- All posts for specific day
- Detailed timing information
- Quick edit and reschedule
- Platform breakdown

### Scheduling Content

**From Calendar:**

1. Click date on calendar
2. Select "Schedule Post"
3. Choose content from library
4. Select platforms
5. Set date and time
6. Customize per platform
7. Schedule

**From Content Library:**

1. Browse ingredients
2. Select content to publish
3. Click "Schedule"
4. Choose platforms
5. Set date and time
6. Customize content
7. Schedule

### Rescheduling

- Drag and drop on calendar
- Click post and change time
- Bulk reschedule multiple posts
- Use "Best Time to Post" recommendations

## Thread Composer

### What Are Threads?

Threads are multi-part posts for Twitter/X that allow you to share longer content by breaking it into connected posts.

### Creating a Thread

**Step 1: Start Thread**

1. Click "Compose Thread"
2. Enter first post content
3. Add media if needed
4. Preview first post

**Step 2: Add Thread Posts**

1. Click "Add Post" to thread
2. Enter next post content
3. Continue adding posts
4. Preview entire thread

**Step 3: Schedule Thread**

1. Set posting time
2. Choose posting interval (seconds between posts)
3. Preview thread order
4. Schedule thread

### Thread Best Practices

- **Hook**: First post should grab attention
- **Flow**: Each post should flow naturally to next
- **Length**: Keep individual posts concise
- **Media**: Use images/videos strategically
- **Timing**: Space posts 30-60 seconds apart

### Thread Templates

- Save successful thread structures
- Reuse thread formats
- Create thread templates for common topics
- Quick compose from templates

## Best Time to Post

### AI-Powered Recommendations

Publisher analyzes your audience engagement data to recommend optimal posting times.

**How It Works:**

1. Analyzes your historical performance
2. Identifies peak engagement times
3. Considers platform-specific patterns
4. Recommends best times for your audience

**Using Recommendations:**

1. Select content to schedule
2. Click "Best Time to Post"
3. View recommended times
4. See predicted engagement scores
5. Schedule at recommended time

### Platform-Specific Timing

**Twitter/X:**

- Peak times vary by audience
- Generally 8-10 AM and 7-9 PM
- Use recommendations for your audience

**Instagram:**

- Best times: 11 AM - 1 PM, 7-9 PM
- Stories: Throughout the day
- Reels: Peak engagement times

**LinkedIn:**

- Best: Tuesday-Thursday, 8-10 AM
- Professional audience hours
- Avoid weekends

**TikTok:**

- Peak: 6-10 AM, 7-9 PM
- Younger audience patterns
- Consistent daily posting

- `⌘3` / `Ctrl+3`: Open Publisher
- `⌘N` / `Ctrl+N`: New scheduled post
- `⌘T` / `Ctrl+T`: Thread composer
- `⌘B` / `Ctrl+B`: Best time to post
- `Space`: Preview scheduled post

## Troubleshooting

### Scheduling Fails

- Check platform connections
- Verify account permissions
- Ensure content meets platform requirements
- Check internet connection

### Best Time Not Available

- Need historical data for recommendations
- Connect platforms to gather data
- Wait for data collection period
- Use general best practices in meantime

### Thread Not Posting

- Check thread length and format
- Verify platform supports threads
- Check posting interval settings
- Ensure account has thread permissions

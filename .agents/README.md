## Task Policy (Important)

GitHub Issues/Projects are the canonical task system.

- Do not treat local `INBOX.md` files as canonical backlog.
- `TASKS/` exists for policy compatibility and scoped local notes only when explicitly requested.

## Dual-State Rule For Status Docs

For roadmap/features/architecture status pages:

- **Implemented State:** what code currently does.
- **Delivery State:** what GitHub project/issues track.

When these differ, document both explicitly.

## Historical Content Policy

`archive/` and `SESSIONS/` are historical records.

- Do not rewrite historical narratives to match current structure.
- Add forward-looking notes in canonical docs instead of retro-editing logs.

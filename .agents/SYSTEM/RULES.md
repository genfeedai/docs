# Tracking Rules (Docs-Site Overlay)

This file supplements the shared workspace canon.

- Shared workspace rules live in [`../../../.agents/SYSTEM/critical/CROSS-PROJECT-RULES.md`](../../../.agents/SYSTEM/critical/CROSS-PROJECT-RULES.md).
- GitHub Issues are the only active source of truth for tasks and PRDs.
- Do not create or maintain local task files under `.agents/TASKS/` (except `README.md`).
- Do not create or maintain local PRD files under `.agents/PRDS/` (except `README.md`).
- Do not keep local task/PRD archives in this repository.
- Do not keep local task snapshots/reports (for example: `issues-all.json`, `AUDIT-SYNC-REPORT.md`, `INBOX.md`).
- For `genfeedai/cloud`, GitHub assignee is the authoritative reservation signal for human-owned work.
- If an issue is assigned to `VincentShipsIt`, agents must exclude it from planning and implementation until it is reassigned.
- Do not use the Mission Control `Owner` field to decide whether an issue is reserved for a human.
- Keep docs-site-local notes here only when they are specific to the docs project.
